# Index in SQL:
- An index is a special data structure that helps the database find rows faster, similar to an index in a book.

### Without an Index:
- Suppose you have 1 million employees and run:
```
SELECT *
FROM Employees
WHERE EmployeeID = 500000;
```
- Without an index, the database may need to check rows one by one:
```
1 → 2 → 3 → 4 → ... → 500000
```
- This is called a Table Scan or Full Scan.
- O(n)

### With an Index:
- If EmployeeID is indexed:
```
CREATE INDEX IX_Employees_EmployeeID
ON Employees(EmployeeID);
```
- nO(logn)

## How Does an Index Optimize Queries:
- Most databases use a B-Tree (Balanced Tree) structure for indexes.
- The database navigates the tree to find the row much faster.
