# Group by clause:

## Syntax:
```
SELECT column1, aggregate_function(column2), column3, ...
FROM table_name
WHERE condition
GROUP BY column1, column3
Having condition
ORDER BY column_name;
```
- Put similar records into buckets, then calculate something for each bucket.
- The GROUP BY clause in SQL is used to group rows that have the same values in specified columns.
- After grouping, you usually apply aggregate functions like COUNT(), SUM(), AVG(), MAX(), MIN() to each group.

```
SELECT column_name, aggregate_function(column_name)
FROM table_name
GROUP BY column_name;
```
- Table: **Employees**

| Id | Name | Department | Salary |
| -- | ---- | ---------- | ------ |
| 1  | A    | HR         | 5000   |
| 2  | B    | IT         | 7000   |
| 3  | C    | IT         | 8000   |
| 4  | D    | HR         | 6000   |

1. Count employees in each department
```
SELECT Department, COUNT(*) AS TotalEmployees
FROM Employees
GROUP BY Department;
```
- Result:
| Department | TotalEmployees |
| ---------- | -------------- |
| HR         | 2              |
| IT         | 2              |

2. Total salary per department
```
SELECT Department, SUM(Salary) AS TotalSalary
FROM Employees
GROUP BY Department;
```

## Important Note:
- Every column in SELECT must be:
  - Either in GROUP BY, OR
  - Inside an aggregate function
- You cannot select a normal column without grouping it.

- ❌ Wrong:
```
SELECT Name, Department
FROM Employees
GROUP BY Department;
```
- Name is not grouped or aggregated.

## GROUP BY with HAVING 
- HAVING filters groups (after grouping)
```
SELECT Department, COUNT(*) AS Total
FROM Employees
GROUP BY Department
HAVING COUNT(*) > 1;
```
